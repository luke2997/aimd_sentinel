# AIMD Sentinel final demo package

This demo contains three sample governance dossiers:

1. UNiD Spine Analyzer
   - Positive software-governance example.
   - 15 high-confidence device-specific adverse-event links.
   - Main themes: software anomaly, calculation/classification disagreement, software version/CAPA-type investigation.

2. Lung Vision System
   - Smaller software/workflow example.
   - 3 high-confidence device-specific adverse-event links.
   - Main themes: synchronization/registration/workflow issues plus possible hardware malfunction.

3. Red Dot
   - Evidence-gap and no-overclaiming example.
   - 0 high-confidence adverse-event links.
   - 4 broad/review enforcement links manually labelled as not device-specific.

Manual validation files:
- final_demo_event_validation_labeled.csv
- final_demo_red_dot_enforcement_validation_labeled.csv

Important limitation:
The dossiers support procurement, governance, and safety-review workflows only. They do not establish causality, incidence rates, or clinical/legal safety conclusions.
